Java多线程

线程与进程

进程是操作系统分配资源的最小单位，线程是程序执行的最小单位。

一个进程有一个或多个线程组成，线程共享进程资源，进程之间则是相互独立。

线程通过直接读写同一进程中的数据进行通信，进程则有自己的通信方式（IPC）

线程切换的开销比进程小。

进程的几种通信方式，进程间通信(IPC)

  • 管道（pipe），半双工，限制在父子/兄弟进程中使用
  • 命名管道（fifo），可用于不同进程间通信，常用于客户-服务器应用程序中
  • 消息队列，可独立于进程存在，避免了fifo的同步阻塞问题
  • 信号量，是一个计数器，用于为多个进程提供对共享数据的访问，和共享存储配合用于同步
  • 共享存储，多个进程共享一个给定的存储区，数据不用在进程之间复制，通信速度最快
  • 套接字，用于不同机器间的进程通信

实现接口和继承Thread类

  • Java不支持多重继承，继承了Thread类则无法继承其他类，接口则可以实现多个
  • 继承整个Thread类开销更大，实现接口则更灵活

线程池

使用线程池的好处如下。

  • 降低资源消耗。通过重复利用已创建的线程降低线程创建和销毁造成的开销。
  • 提高响应速度。当任务到达时可以不用等线程创建就能执行。
  • 提高线程的可管理性。使用线程池可以进行统一分配、调优和监控。

常用线程池

首先有两个线程池框架，ThreadPoolExecutor和ScheduledThreadPoolExecutor

ThreadPoolExecutor主要由以下4个组件构成

  • corePool：核心线程池的大小
  • maximumPool：最大线程池的大小
  • BlockingQueue：用于暂时保存任务的工作队列
  • RejectedExecutorHandler：关闭或饱和（达到了最大线程池大小且工作队列已满）时采用的拒绝策略

通过Executor框架的工具类Executors，可以创建3种封装好的ThreadPoolExecutor

  • FixedThreadPool，创建固定数量的线程

    java

  • SingleThreadExecutor，相当于大小为1的FixedThreadPool

    java

  • CachedThreadPool，根据需要创建新线程的线程池

    java

ScheduledThreadPoolExecutor则继承自ThreadPoolExecutor，通过DelayQueue（封装了一个按执行时间排序的优先队列），延期或者定时执行任务。Executors工具类可以创建2种ScheduledThreadPoolExecutor。

  • ScheduledThreadPoolExecutor。包含若干个线程。
  • SingleThreadScheduledExecutor。只包含一个线程。
